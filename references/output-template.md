## 🔍 字段填写规范

### 子景点数量
- **最少**：3 个（小型景点如古镇）
- **推荐**：5-6 个（城市级方案）
- **最多**：9 个（省级大盘如四川 8 大景点可各占一席）

### 类型覆盖检查
每个方案应至少覆盖以下 4 种类型中的 3 种：

- [ ] **自然风光**（山 / 水 / 林 / 草原 / 海）
- [ ] **人文古迹**（寺庙 / 古镇 / 遗址 / 建筑）
- [ ] **美食体验**（当地代表食物 / 老字号 / 夜市）
- [ ] **季节限定**（樱花 / 红叶 / 雪景 / 花海）
- [ ] **夜生活**（夜景 / 夜市 / 酒吧 / 演出）
- [ ] **小众 / 在地**（隐藏玩法 / 本地人推荐）

### 文案参考长度
- 标题：12-18 字（不超过 20）
- 正文：30-80 字（在卡片信息密度内 1 屏可读完）
- AI 提示词：30-60 字英文（不超过 80）

### 实用信息颗粒度
| 字段 | 必填 | 备注 |
|---|---|---|
| 门票 | ✅ | **必须**给具体数字或区间 |
| 开放时间 | ✅ | 季节性差异需标注 |
| 最佳季节 | ✅ | 写月份区间 |
| 游玩时长 | ✅ | 区间格式 `x-x 小时` |
| 交通 | ⚠️ | 公共交通 + 自驾各 1 条 |
| 人均花费 | ⚠️ | 仅当预算明确时填 |
| 地址 | ⚠️ | 知名景点必填 |

---

## ✅ 自检清单（输出前必过）

发布前请检查以下 7 项，全部 ✅ 才能算合格输出：

- [ ] **1. 结构完整**：8 个标准章节都有，无遗漏
- [ ] **2. 子景点 ≥3**：景点详细方案至少包含 3 个
- [ ] **3. 类型覆盖**：自然 / 人文 / 美食 / 季节 至少 3 种
- [ ] **4. 文案 ≥3 种风格**：系列打卡 + 情绪共鸣 + 反差吸引 必备
- [ ] **5. AI Prompt 有参数**：每个 prompt 必须带 `--ar / --v`
- [ ] **6. 实用信息有数字**：所有价格 / 时间 / 距离必须是具体数字
- [ ] **7. 引用来源 ≥3 条**：参考链接不能是「待补充」

未达标 → 重新生成对应章节，而不是整体重写。

## 📏 字数预算

| 章节 | 推荐字数 |
|---|---|
| 目的地简述 | 60-100 |
| 单个子景点 | 200-350 |
| 发布建议 | 150-250 |
| 热门趋势 | 100-180 |
| 参考来源 | 50-100 |
| **合计** | **1500-2500**（不含 Prompt 与表格）|

> **如果输出超过 3000 字**：考虑拆分为「概览篇（1500 字）」+ 「分景点详解篇（每个独立）」，方便用户分开发布。

---

## 🧬 模板可裁剪

对于「只想发朋友圈」的轻量用户，可以裁剪掉以下章节：

```
✅ 必保留：目的地简述 + 单个子景点 + 标题 + 标签
🟡 可选保留：实用信息速查表 + 视觉元素特写
❌ 可裁掉：完整 AI prompt 长版 + 热门趋势 + 参考来源
```

## 📂 命名与归档

建议按以下方式保存输出：

```
examples/
├── chengdu-output.md
├── leshan-output.md
├── yibin-output.md
├── sichuan-output.md
└── hokkaido-2026-winter-output.md   # 自定义
```

文件名建议：
- **小写 + 英文**（避免中文文件名在不同系统的问题）
- **`<景点>-output.md`** 统一后缀
- 涉及时间敏感的内容可加日期
